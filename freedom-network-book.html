Preface

Why This Book Exists

The political and technical case for a people's internet.

Every authoritarian playbook includes the same move: control the communications infrastructure. Kill the cell towers, throttle the ISPs, block the DNS resolvers, log the packet flows, arrest the administrators. History rhymes — from the Soviet jamming of BBC broadcasts to the Myanmar military cutting internet during the 2021 coup, from the Iranian internet blackouts during protests to the Chinese Golden Shield filtering every packet leaving the country.

The answer is not to ask governments for permission to communicate. The answer is to build infrastructure that inherently resists centralized control — networks that route around damage, encrypt by default, require no central registry, and can be deployed with commodity hardware and open-source software by ordinary people.

This is a technical handbook for doing exactly that. We begin in a Proxmox homelab — a safe sandboxed environment where you can spin up a dozen virtual machines and simulate an entire mesh city — and we provide every configuration file, command, and explanation you need. The same configs run in the real world.

Yggdrasil — The IPv6 Overlay Mesh

An end-to-end encrypted, self-addressing network where every node gets a cryptographically derived IPv6 address and routes without any central authority.

What Is Yggdrasil?

Yggdrasil is a next-generation overlay network built on a spanning-tree-like routing protocol where addresses are derived from each node's public key. There is no DHT, no central directory, and no single point of failure. Every packet is end-to-end encrypted using ephemeral Curve25519/ChaCha20-Poly1305.

Unlike I2P or Tor which are designed for anonymity, Yggdrasil is designed for resilient global reachability. It can run over:

  • TCP/UDP over the existing internet (peering)
  • Direct Ethernet / WiFi links (no IP layer needed)
  • batman-adv mesh networks (our primary use case)
  • LoRa / serial links (with appropriate transport glue)

Address Derivation

Every node generates an Ed25519 keypair. The public key is hashed using SHA-512 and the result is used to derive a 200::/7 IPv6 address. This means:

  • No DHCP server is needed — addresses are self-sovereign
  • Any address you talk to, you are cryptographically certain of the peer's identity
  • Adding a new node to the network requires zero configuration from other nodes

📄 mesh-router-1, mesh-router-2, mesh-router-3 — install Yggdrasil

Configuration

Yggdrasil generates a config file with a fresh keypair on first run. We then add peers and optionally expose multicast discovery on local Ethernet links.

📄 /etc/yggdrasil/yggdrasil.conf — mesh-router-1
{
  Peers: [
    // Peer with router-2 via direct Ethernet (vmbr1 addresses)
    "tcp://10.10.1.2:9001",
    "tcp://10.10.1.3:9001"
  ]

  Listen: [
    "tcp://0.0.0.0:9001"
  ]

  // Multicast enables automatic discovery on local Ethernet/WiFi
  MulticastInterfaces: [
    {
      Regex: "eth.*|ens.*|wlan.*|bat0"
      Beacon: true
      Listen: true
      Port: 0
      Priority: 0
    }
  ]

  IfName: "ygg0"
  IfMTU: 65535

  // Admin socket for yggdrasilctl
  AdminListen: "unix:///var/run/yggdrasil/yggdrasil.sock"

  // Keys auto-generated on first start — do not share PrivateKey
  PrivateKey: "<auto-generated>"
}
📄 Start and enable
systemctl enable --now yggdrasil

# Verify your Yggdrasil IPv6 address
yggdrasilctl getSelf
# Should show something like: address: 200:1234:abcd:...

# Ping router-2 over Yggdrasil
ping6 200:xxxx:yyyy:zzzz::1

Integrating with batman-adv

The most powerful setup runs Yggdrasil on top of batman-adv. batman-adv creates a virtual bat0 Ethernet interface — Yggdrasil's multicast discovery then finds peers automatically across any physical medium batman-adv supports (WiFi, Ethernet, LoRa bridges). 

# After bat0 is up (see Chapter 2), ensure Yggdrasil listens on it
# The MulticastInterfaces regex "bat0" above handles this automatically.
ip link show bat0
# Yggdrasil peers will appear when routers can reach each other
yggdrasilctl getPeers

Public Yggdrasil Network

Yggdrasil has a growing global network of peers. By adding one or more public peers to your Peers list, your homelab nodes immediately join the global mesh, accessible at their 200:: address from anywhere in the world that runs Yggdrasil — without NAT, without port forwarding, without a VPS.

📄 Add public peers to yggdrasil.conf
Peers: [
  // Community peers — see https://publicpeers.neilalexander.dev/
  "tls://s1.nl.yggo.net:443",

]
Real-World Deployment Each LoRa node, each Raspberry Pi in a car, each apartment WiFi device — once it runs Yggdrasil, it gets a permanent globally routable IPv6 address that never changes regardless of what ISP or physical medium it is on. Services (Matrix, XMPP, IPFS) bind to this address and are reachable from anywhere in the Yggdrasil network.

Verification Checklist

  • Each router shows a 200:: address in yggdrasilctl getSelf
  • Routers can ping each other's Yggdrasil addresses
  • Multicast discovery works on the shared segment (no manual peer config needed)
  • Services (Matrix later) can bind to :: and be reached via Yggdrasil
Chapter 2

batman-adv — The Layer 2 Mesh Foundation

Better Approach To Mobile Adhoc Networking — Advanced. A kernel module that turns any collection of WiFi (or Ethernet) interfaces into a self-healing bridged mesh.

How batman-adv Works

batman-adv operates at Layer 2 (Ethernet), not Layer 3. It wraps every node's physical interfaces into a virtual bat0 Ethernet interface. From the perspective of higher layers (IP, Yggdrasil), bat0 looks like a simple switch connected to all other mesh nodes — the routing complexity is hidden inside the kernel module.

Each node broadcasts OGMs (Originator Messages) — small beacon frames — at configurable intervals. Every node tracks which neighbor it should forward packets through to reach every other node, continuously re-evaluating as links appear and disappear. A node leaving (router reboots, jammed link) is rerouted around within seconds.

FeatureDetail
ProtocolBATMAN IV / V (version selectable)
TransportAny Ethernet-compatible medium
Routing metricTQ (Transmission Quality) 0–255
Kernel modulebatman-adv in Linux kernel since 3.2
Convergence~1–3 s for single link failure

Configuration — Ethernet Simulation (Proxmox)

In Proxmox, each mesh VM has a second NIC (ens19) connected to vmbr1. We configure batman-adv to treat this Ethernet link as a mesh interface.

📄 /etc/network/interfaces — mesh-router-1
# Primary management interface
auto ens18
iface ens18 inet dhcp

# Mesh interface — no IP, batman-adv handles it
auto ens19
iface ens19 inet manual
    pre-up ip link set ens19 promisc on mtu 1532
    up batctl if add ens19
    up ip link set bat0 up
    up ip addr add 10.10.1.1/24 dev bat0   # management IP on mesh
    down batctl if del ens19
📄 /etc/batctl.conf (optional tuning)
# Use BATMAN V (metric: throughput) instead of IV (metric: hop count)
routing_algo BATMAN_V
# OGM interval in milliseconds (default 1000)
orig_interval 1000
📄 Bring up the mesh and verify
systemctl restart networking

# Show originators (other mesh nodes)
batctl o

# Show routing table
batctl n

# Show link quality to each neighbor (TQ out of 255)
batctl tl

# Ping router-2 via mesh
ping 10.10.1.2

On real hardware with a WiFi adapter, configure the adapter in 802.11s mesh or ad-hoc (IBSS) mode, then add it to batman-adv.

Multi-Hop Routing Demo

With three mesh routers (A, B, C) where A and C cannot see each other but both see B, batman-adv automatically routes A→C via B. Remove VM 101 (B) and watch batman-adv converge a direct route if A and C gain direct visibility, or declare C unreachable if not. 

# On mesh-router-1, watch routing table change when router-2 goes down
watch -n 1 "batctl o; echo; batctl n"

# In another terminal, stop router-2 in Proxmox

# Observe: within ~3 seconds, router-1 either finds alt route or drops router-3

batman-adv + Yggdrasil Together

Once bat0 is up and peers are discovered, Yggdrasil's multicast layer auto-discovers Yggdrasil peers on the same bat0 segment. You now have:

  • batman-adv handling Layer 2 mesh (self-healing Ethernet fabric)
  • Yggdrasil handling end-to-end encrypted routable IPv6 on top
  • Every node addressable by a stable cryptographic IPv6 regardless of physical path

Matrix — Federated Encrypted Messaging

An open standard for real-time communication. Each homeserver is independently operated; rooms are replicated across every server whose users participate.

Why Matrix?

Matrix's federation model means that destroying one server does not destroy any room — every participating server holds a complete copy of the room's event graph. End-to-end encryption (Megolm / Olm) ensures that even a compromised server cannot read messages. The protocol supports text, voice, video, files, and bridges to legacy systems (IRC, XMPP, Telegram, Signal).

Nostr — Protocol, Not Platform

Notes and Other Stuff Transmitted by Relays. A minimalist cryptographic social protocol with no central registry, no accounts — only keypairs.

Nostr Philosophy

Nostr is deliberately minimal. There are no servers — only relays. There are no accounts — only Ed25519 keypairs. Your identity is your public key. A relay stores and serves signed events (messages, posts, follows, DMs). Any relay can serve any event signed by any key. Censor one relay and the event lives on every other relay that has it.

This makes Nostr extremely resistant to deplatforming: you cannot ban a keypair because keys are mathematical objects, not accounts on a server.

Connecting Clients

Use any Nostr client (Iris, Snort, Coracle, Nostrudel) and add ws://nostr-relay.mesh.local:7777 as a relay. Generate a keypair in the client — your npub (public key) is your identity, your nsec (private key) must be guarded at all costs.

⚠️ Key Security Never paste your nsec into a browser extension or web client you don't trust. Use a hardware signer (e.g., a dedicated offline device with Alby or Amber) and only sign with the private key locally. The relay never sees your private key.

Nostr Direct Messages (NIP-04 / NIP-17)

Nostr supports encrypted DMs via NIP-17 (Gift Wrap). Messages are double-encrypted: once to the recipient's public key, and once wrapped with a random one-time key, so relays cannot see who is messaging whom — only the recipient can unwrap.

Multiple Relays = Censorship Resistance

Deploy strfry on three nodes (VM 111, 112-relay, mesh-router-3). Configure your client to publish to all three. Any single relay going down — or a legal takedown notice — leaves your posts available on the others.

Asterisk — Free Voice Over IP

An open-source PBX and SIP server that enables encrypted voice and video calls across the mesh network — no PSTN, no cellular, no central provider.

Why Asterisk in a Freedom Network?

Phone calls are the highest-value communication in a crisis. When a cell network is blocked or a mesh node has limited text bandwidth, a quick encrypted voice call conveys more information per byte than any text protocol. Asterisk provides:

  • SIP-based VoIP — works over any IP network including the mesh
  • WebRTC support — browser-based calls, no app install needed
  • SRTP/DTLS encryption — media encrypted in flight
  • TLS signalling — SIP messages encrypted
  • Zero dependency on PSTN or any telco infrastructure

Hyphanet — Censorship-Resistant Datastore

Formerly Freenet. A peer-to-peer network where data is stored across thousands of nodes, encrypted and distributed — no single node holds complete files, and no single node knows who requested or published anything.

Hyphanet's Model

Hyphanet operates on a principle of plausible deniability. When you store a file, it is split, encrypted, and distributed across the network. No node knows what data it stores (it's encrypted), who inserted it (routing is anonymous), or who is retrieving it. A node that receives a request for data it doesn't have forwards the request to another node — like a blind relay race.

Two modes of operation:

  • Opennet — connects to random peers. Easier to start, less privacy.
  • Darknet — connects only to friends (trusted peers with known keys). Maximum privacy; must exchange node references out-of-band.

Key Types — CHK, SSK, and USK

IPFS — Content-Addressed Distributed Storage

InterPlanetary File System. Every file is identified by the hash of its content — anyone who has it can serve it, and you always get exactly what you asked for.

Content Addressing vs. Location Addressing

On the traditional web, a URL identifies where a file is stored. If that server goes down, the file is gone. With IPFS, a CID (Content Identifier) is the hash of the file's content. You retrieve the content from whoever has it — any node in the network. The content is cryptographically verified against the CID.

  • A government blocks a URL → irrelevant, the same CID is served from thousands of nodes
  • A hosting provider deletes content → irrelevant if anyone has pinned the CID
  • A file is tampered with → immediately detectable (hash mismatch)

Software-Defined Radio running spread-spectrum waveforms that are hard to detect, impossible to selectively block, and resilient to deliberate jamming by an adversary with significant resources.

Why Spread Spectrum?

A conventional narrowband radio transmits all its power in a narrow slice of spectrum. A jammer needs only match that slice to destroy the link. Spread spectrum techniques deliberately spread the signal across a wide swath of spectrum, so any given frequency slice carries almost no power — the signal looks like noise. Two families:

TechniqueHow it worksPrimary strength
DSSS
Direct Sequence Spread Spectrum		 XOR the data bits with a high-rate pseudo-random "chip" sequence. Spreads signal across the full bandwidth at once. Processing gain (SNR recovery), covert operation, hard to jam
FHSS
Frequency Hopping Spread Spectrum		 Transmitter rapidly hops between frequencies in a pseudo-random sequence shared with the receiver. Anti-selective-jamming, hard to intercept, harder to detect
CSS
Chirp Spread Spectrum (LoRa)		 Frequency sweeps linearly across bandwidth during each symbol. Covered in Chapter 2 appendix. Extreme range at low power, Doppler immune

DSSS Implementation in GNU Radio

DSSS multiplies each data bit by a spreading code (a Pseudo-Noise / PN sequence). If the chip rate is N times the bit rate, we gain 10·log₁₀(N) dB of processing gain — meaning the receiver can pull the signal out of noise that would bury it completely for an eavesdropper without the code.

📄 Python — Generate a maximal-length PN spreading sequence
import numpy as np

def lfsr_sequence(taps, length):
    """
    Generate a maximal-length LFSR sequence (m-sequence).
    taps: feedback tap positions (e.g., [7, 6] for a 127-chip m-sequence)
    """
    n = max(taps)
    state = [1] * n
    seq = []
    for _ in range(length):
        out = state[-1]
        seq.append(out)
        feedback = sum(state[t - 1] for t in taps) % 2
        state = [feedback] + state[:-1]
    return np.array(seq, dtype=np.float32) * 2 - 1   # map 0/1 → -1/+1

# 127-chip m-sequence (spreading factor = 127)
pn_code = lfsr_sequence([7, 6], 127)
print(f"Processing gain: {10 * np.log10(len(pn_code)):.1f} dB")
# → Processing gain: 21.0 dB  (signal recoverable 21 dB below noise floor)

class DSSSTransmitter(gr.top_block):
    def __init__(self):
        gr.top_block.__init__(self)

        # Source: read data from stdin as packed bytes
        self.src = blocks.file_descriptor_source(gr.sizeof_char, 0, True)
        self.pack = blocks.packed_to_unpacked_bb(1, gr.GR_MSB_FIRST)

        # Spread: XOR each bit with the repeating PN code
        pn_src = blocks.vector_source_f(pn_code.tolist(), True)
        self.spread = blocks.multiply_ff()

        # Convert to complex (BPSK modulation)
        self.to_float = blocks.char_to_float(1, 1)
        self.add_const = blocks.add_const_ff(-0.5)
        self.scale     = blocks.multiply_const_ff(2.0)
        self.to_complex = blocks.float_to_complex()

        # Pulse shaping
        taps = filter.firdes.root_raised_cosine(
            1, SAMP_RATE, CHIP_RATE, 0.35, 64)
        self.rrc = filter.interp_fir_filter_ccf(
            int(SAMP_RATE / CHIP_RATE), taps)

        self.sink = osmosdr.sink()
        self.sink.set_sample_rate(SAMP_RATE)
        self.sink.set_center_freq(CENTER_FREQ)
        self.sink.set_gain(GAIN)

        # Connect
        self.connect(self.src, self.pack, self.to_float,
                     self.add_const, self.scale)
        self.connect(self.scale, (self.spread, 0))
        self.connect(pn_src,    (self.spread, 1))

FHSS Implementation

FHSS requires the transmitter and receiver to hop through the same sequence of frequencies at the same time, derived from a shared secret seed. A jammer must transmit on all frequencies simultaneously to block every hop — requiring power proportional to the number of channels.

📄 fhss_controller.py — Frequency hop sequence generator
import hashlib, struct, time

class FHSSController:
    """
    Generates a deterministic pseudo-random hop sequence from a shared secret.
    Both TX and RX must share: secret, hop_channels, dwell_time_ms
    """

    def __init__(self, secret: bytes, hop_channels: list[float], dwell_ms: float = 10):
        self.secret   = secret
        self.channels = hop_channels   # list of center frequencies in Hz
        self.dwell    = dwell_ms / 1000

    def current_channel(self) -> float:
        """Return the channel we should be on right now."""
        slot = int(time.time() / self.dwell)
        h = hashlib.sha256(self.secret + struct.pack(">Q", slot)).digest()
        idx = int.from_bytes(h[:4], "big") % len(self.channels)
        return self.channels[idx]

# ISM 902–928 MHz divided into 50 channels × 512 kHz
channels = [902.25e6 + i * 512e3 for i in range(50)]

controller = FHSSController(
    secret = b"freedom-net-psk-2025",
    hop_channels = channels,
    dwell_ms = 5   # hop every 5 ms → 200 hops/second
)

# In a thread: retune SDR hardware to current_channel() every dwell_ms
import threading

def hop_loop(sdr_sink, ctrl):
    while True:
        freq = ctrl.current_channel()
        sdr_sink.set_center_freq(freq)
        time.sleep(ctrl.dwell)

t = threading.Thread(target=hop_loop, args=(sink, controller), daemon=True)
t.start()

Reticulum — Cryptographic Networking for the Edge

A cryptography-based networking stack designed to work reliably on high-latency, low-bandwidth, and intermittently connected links — LoRa, packet radio, I2P, TCP, and serial — with built-in identity, encryption, and routing.

What Makes Reticulum Different?

Most networking stacks assume reliable, low-latency links and add encryption as an afterthought. Reticulum inverts this: it is built ground-up for challenged links and treats cryptography as the foundation of addressing and routing.

  • Every identity (destination) is a truncated SHA-256 hash of an Ed25519 public key — no DNS, no registry, no CA
  • Packets are encrypted with X25519 ECDH (forward secrecy) + AES-128-CBC + HMAC-SHA-256
  • Routing is opportunistic — nodes announce reachability, and paths are discovered lazily
  • Works over serial at 1200 baud, LoRa at 250 bps, or fiber at 10 Gbps — the same stack
  • Built-in store-and-forward for disconnected operation (like UUCP but encrypted)

Core Concepts

Identities and Destinations

An Identity is an Ed25519 + X25519 keypair. A Destination is a named endpoint derived from one or more identities and an application name string. Destinations are 16-byte truncated hashes — short enough for LoRa packets, long enough to be collision-resistant.

Links

A Link is an established encrypted session between two destinations. Link setup performs a Diffie-Hellman key exchange over potentially many hops, establishes a shared session key, and then provides a reliable-ish encrypted channel regardless of the underlying transport's reliability.

LXMF — Lightweight Extensible Message Format

LXMF is Reticulum's built-in messaging protocol. Messages are propagated store-and-forward through the network — if the destination is offline, the message is held by intermediate nodes until delivered. Think of it as encrypted SMTP designed for LoRa.

Distributed Hash Tables — The Directory of Everything

The data structure that powers IPFS content routing, I2P's NetDB, Yggdrasil's path discovery, Nostr relay lists, and countless other decentralised systems. Understanding DHTs lets you build — and harden — any of them.

What Is a DHT?

A Distributed Hash Table is a key-value store spread across many nodes with no central coordinator. Each node is responsible for storing values whose keys fall within its portion of a keyspace. Looking up a key involves asking a small number of nodes (typically O(log N)) in a chain of referrals until you reach the node responsible for that key.

In a freedom network, DHTs provide:

  • Peer discovery — find nodes without a directory server
  • Content routing — find who holds a given IPFS CID
  • Service advertisement — announce hidden services without a central registrar
  • Name resolution — resolve human-readable names to addresses without DNS

Kademlia — The Dominant DHT Algorithm

Most real-world DHTs (IPFS, BitTorrent, I2P, Ethereum) use Kademlia or a variant. Its key insight: nodes and keys share the same keyspace (usually 160 or 256 bits), and distance between IDs is measured with XOR, not physical distance. XOR distance has a useful property: for any node A and any key K, there exists a unique node closest to K in every sub-tree, enabling logarithmic lookup convergence.

Implementing a Simple Kademlia Node in Python

📄 kademlia_node.py — Core data structures
import hashlib, os, asyncio
from typing import Optional

ID_BITS  = 256
K        = 20    # bucket size
ALPHA    = 3     # lookup parallelism

class NodeID:
    def __init__(self, raw: bytes = None):
        self.raw = raw or os.urandom(32)   # random 256-bit ID on first boot

    def distance(self, other: "NodeID") -> int:
        a = int.from_bytes(self.raw, "big")
        b = int.from_bytes(other.raw, "big")
        return a ^ b

    def bucket_index(self, other: "NodeID") -> int:
        d = self.distance(other)
        return d.bit_length() - 1   # 0..255 which k-bucket does 'other' fall into

    def __repr__(self):
        return self.raw.hex()[:16] + "..."

class KBucket:
    def __init__(self):
        self.nodes: list = []     # list of (NodeID, addr) tuples, LRU order

    def add(self, node_id, addr):
        existing = [n for n in self.nodes if n[0].raw == node_id.raw]
        if existing:
            self.nodes.remove(existing[0])
            self.nodes.append((node_id, addr))   # move to tail (most recent)
        elif len(self.nodes) < K:
            self.nodes.append((node_id, addr))
        else:
            # Bucket full: ping head (oldest); if dead, replace. If alive, drop new.
            pass   # simplified — real impl pings head[0]

    def closest_to(self, target: NodeID, n: int = K) -> list:
        return sorted(self.nodes, key=lambda x: x[0].distance(target))[:n]

class RoutingTable:
    def __init__(self, local_id: NodeID):
        self.local  = local_id
        self.buckets = [KBucket() for _ in range(ID_BITS)]

    def add_node(self, node_id: NodeID, addr):
        idx = self.local.bucket_index(node_id)
        self.buckets[idx].add(node_id, addr)

    def find_closest(self, target: NodeID, n: int = K) -> list:
        idx = self.local.bucket_index(target)
        # Check idx ± surrounding buckets until we have n nodes
        results = []
        for offset in range(ID_BITS):
            for i in [idx - offset, idx + offset]:
                if 0 <= i < ID_BITS:
                    results.extend(self.buckets[i].closest_to(target, n))
            if len(results) >= n:
                break
        return sorted(results, key=lambda x: x[0].distance(target))[:n]
📄 kademlia_node.py — Iterative lookup coroutine
class KademliaNode:
    def __init__(self):
        self.id    = NodeID()
        self.table = RoutingTable(self.id)
        self.store: dict[str, bytes] = {}   # local key-value store

    async def lookup(self, key: NodeID) -> list:
        """Iterative lookup — find the k nodes closest to 'key'."""
        closest  = self.table.find_closest(key, ALPHA)
        visited  = set()
        result   = list(closest)

        while True:
            batch = [n for n in result if n[0].raw not in visited][:ALPHA]
            if not batch:
                break

            tasks = [self._send_find_node(addr, key) for _, addr in batch]
            responses = await asyncio.gather(*tasks, return_exceptions=True)

            improved = False
            for (node_id, _), resp in zip(batch, responses):
                visited.add(node_id.raw)
                if isinstance(resp, list):
                    for new_node in resp:
                        if new_node not in result:
                            result.append(new_node)
                            improved = True

            result = sorted(result, key=lambda x: x[0].distance(key))[:K]
            if not improved:
                break

        return result

    async def put(self, key: str, value: bytes):
        """Store value on the k closest nodes to sha256(key)."""
        key_id = NodeID(hashlib.sha256(key.encode()).digest())
        nodes  = await self.lookup(key_id)
        tasks  = [self._send_store(addr, key_id, value) for _, addr in nodes]
        await asyncio.gather(*tasks)

    async def get(self, key: str) -> Optional[bytes]:
        """Retrieve value from the network."""
        key_id = NodeID(hashlib.sha256(key.encode()).digest())
        nodes  = await self.lookup(key_id)
        for _, addr in nodes:
            val = await self._send_find_value(addr, key_id)
            if val is not None:
                return val
        return None

DHT Use Cases in the Freedom Network

1. Decentralised Service Discovery

Instead of a central DNS server, nodes announce their services (Matrix homeserver, IPFS node, XMPP server) into the DHT. Any new node joining the mesh can discover all services by querying the DHT — no configuration file, no service registry.

2. Nostr Relay Discovery

Nostr clients currently hard-code relay URLs. Integrate a DHT layer: every relay publishes its WebSocket URL and public key under a well-known DHT key (sha256("freedom-network-nostr-relays")). Clients query the DHT at startup to discover all live relays automatically.

3. IPFS Content Routing

IPFS's Kademlia DHT (called the DHT Provider Store) maps CIDs to peer IDs. When you pin a CID, your IPFS node announces PROVIDE(CID, yourPeerID) to the DHT. When someone requests the CID, they query the DHT for provider records, then connect directly to providers. No tracker, no central index.

4. Yggdrasil Path Caching

Yggdrasil's source routing can be augmented with a DHT layer (in research builds) to cache known good paths between distant nodes, reducing the tree-climb latency on the global Yggdrasil network.

DHT Security — Attacks and Defences

AttackDescriptionDefence
Sybil Adversary floods DHT with many malicious node IDs near a target key, monopolising lookups Require proof-of-work or proof-of-stake for node ID assignment; use signed node IDs (Reticulum does this)
Eclipse Adversary fills your routing table with malicious peers, controlling all your outgoing queries Diversify bootstrap peers; verify responses; use multiple independent lookups
Poisoning Malicious nodes return incorrect values for FIND_VALUE queries Sign values with producer's key; verify signature on receipt (IPFS CIDs are self-verifying)
Enumeration Walk the DHT to discover all node IDs and addresses Use cryptographic node IDs derived from keypairs; don't publish IP; use I2P as transport
DDoS Flood individual nodes responsible for popular keys Rate limiting; STORE only from authenticated nodes; replicate across more nodes (increase k)

Running a DHT Bootstrap Node in Proxmox

📄 bootstrap_node.py — Persistent DHT entry point for the mesh
from kademlia.network import Server
import asyncio, logging, json, os

logging.basicConfig(level=logging.INFO)

STATE_FILE = "/var/lib/freedom-dht/state.json"

async def main():
    node = Server()
    await node.listen(5678)

    # Load persisted routing table so we don't need to re-bootstrap after restart
    if os.path.exists(STATE_FILE):
        with open(STATE_FILE) as f:
            state = json.load(f)
        await node.bootstrap([(p["host"], p["port"]) for p in state["peers"]])
    else:
        print("No state — this is the first bootstrap node. Others should peer with us.")

    # Pre-populate well-known service keys
    services = {
        "matrix":  "http://matrix-srv.mesh.local:8448",
        "nostr":   "ws://nostr-relay.mesh.local:7777",
        "ipfs":    "/ip4/10.10.1.6/tcp/4001",
        "xmpp":    "xmpp.mesh.local",
    }
    for k, v in services.items():
        await node.set(k, v)
        print(f"Announced: {k} → {v}")

    # Save state every 60 s
    while True:
        await asyncio.sleep(60)
        neighbours = node.protocol.router.find_neighbors(node.node)
        state = {"peers": [{"host": n.ip, "port": n.port} for n in neighbours]}
        os.makedirs(os.path.dirname(STATE_FILE), exist_ok=True)
        with open(STATE_FILE, "w") as f:
            json.dump(state, f)

asyncio.run(main())

Dead-Man Switch

A dead-man switch automatically triggers a protective action (key deletion, partition wipe, alert broadcast) if an operator fails to check in within a defined window. This protects against silent seizure where the operator is unable to trigger manual countermeasures.

📄 dead_man.py — Encrypted check-in with auto-wipe on timeout
import time, subprocess, os, hashlib, hmac, sys
from datetime import datetime

CHECKIN_FILE  = "/run/freedom-checkin"   # RAM-only (tmpfs)
TIMEOUT_HOURS = 24
SECRET        = os.environ.get("DMS_SECRET", "").encode()
TARGETS       = ["/etc/wireguard/server.key", "/etc/yggdrasil/yggdrasil.conf"]

def checkin(token: str):
    expected = hmac.new(SECRET, token.encode(), hashlib.sha256).hexdigest()
    # In practice: client POSTs HMAC-signed timestamp token to a local HTTP endpoint
    with open(CHECKIN_FILE, "w") as f:
        f.write(str(time.time()))
    print(f"[{datetime.now():%H:%M}] ✓ Check-in recorded")

def monitor():
    while True:
        time.sleep(300)   # check every 5 minutes
        if not os.path.exists(CHECKIN_FILE):
            last = 0
        else:
            with open(CHECKIN_FILE) as f:
                last = float(f.read())
        elapsed_hours = (time.time() - last) / 3600

        if elapsed_hours > TIMEOUT_HOURS:
            print("⚠️  Dead-man timeout! Initiating countermeasures.")
            trigger()

def trigger():
    # 1. Shred sensitive key files
    for path in TARGETS:
        if os.path.exists(path):
            subprocess.run(["shred", "-uzn", "7", path])
            print(f"Shredded: {path}")
    # 2. Broadcast alert to Nostr relay and Matrix room
    subprocess.run(["python3", "/opt/freedom/alert_broadcast.py",
                    "DEAD-MAN TRIGGERED — node may be compromised"])
    # 3. Optional: trigger LUKS header wipe (irreversible!)
    # subprocess.run(["cryptsetup", "erase", "/dev/sda2"])
    sys.exit(1)

if __name__ == "__main__":
    if sys.argv[1:] == ["checkin"]:
        checkin(sys.argv[2])
    else:
        monitor()

Day-0 Operator Meeting — The Keysigning Ceremony

Before any hardware is deployed, the founding operators meet in person. This is the single highest-value security event in the life of a community network — every subsequent out-of-band verification traces back to trust established here.

DAY-0 KEYSIGNING AGENDA (2–3 hours, in person, phones off) Attendees: 3–7 founding operators (each brings their own laptop, no borrowed hardware) 1. IDENTITY VERIFICATION (30 min) Each operator presents two forms of physical ID. Community members who know each other vouch for newcomers. Record legal names in an encrypted vault (Ansible vault, not a spreadsheet). 2. KEY GENERATION (45 min) Each operator generates on their own machine: - WireGuard keypair: wg genkey | tee priv.key | wg pubkey > pub.key - SSH keypair (ed25519): ssh-keygen -t ed25519 -C "operator@westside-mesh" - GPG keypair (RSA-4096 or Ed25519): gpg --gen-key NOBODY SHARES PRIVATE KEYS. EVER. 3. KEY EXCHANGE (30 min) Each operator reads their public keys aloud while others verify hash fingerprints. WireGuard pubkey: wg pubkey < priv.key | base64 | sha256sum → spoken aloud SSH fingerprint: ssh-keygen -lf pub.key → spoken aloud GPG fingerprint: gpg --fingerprint → shown on screen 4. PRE-SHARED KEY DISTRIBUTION (30 min) Generate PSKs for each peer pair (see Ch 7) on one laptop, write to paper, hand paper directly to the other operator. Never send PSKs over any network. 5. VAULT BOOTSTRAP (30 min) One operator boots the Ansible vault: ansible-vault create group_vars/all/vault.yml Pastes all public keys. Vault password is split via Shamir's Secret Sharing (ssss-split -t 2 -n 3) — any 2 of 3 operators can reconstruct it. After this meeting: every operator can SSH to every future node, every node trusts every operator's WireGuard key, and the vault is unlockable without requiring all operators to be present simultaneously.

"""
Dynamic Ansible inventory: reads node registry from the DHT and generates

import asyncio, json, sys
from kademlia.network import Server

async def get_inventory():
    node = Server()
    await node.listen(0)   # ephemeral port
    await node.bootstrap([("10.10.1.1", 5678)])

    registry = json.loads(await node.get("metro-node-registry") or "{}")
    return {
        "mesh_routers": {
            "hosts": [n for n, d in registry.items() if d["role"] == "router"],
        },
        "services": {
            "hosts": [n for n, d in registry.items() if d["role"] == "service"],
        },
        "_meta": {
            "hostvars": {
                n: {"ansible_host": d["yggdrasil_addr"], "ansible_user": "admin"}
                for n, d in registry.items()
            }
        }
    }

Phase 2 — Radio Frequency Interference (Jamming)

The SLA deploys broadband jammers to disrupt WiFi and LoRa communications. Military and police jammers can cover entire frequency bands.

JAMMING ATTACK AND COUNTERMEASURES SLA jammer: broadband noise, 900 MHz – 6 GHz ├── Covers 2.4 GHz WiFi → Our counter: FHSS (50 channels, 200 hops/s) ├── Covers 5.8 GHz WiFi → Our counter: Frequency agility + directional antennas ├── Covers 915 MHz LoRa → Our counter: DSSS processing gain (21 dB) + hop to 433 MHz └── Covers 433 MHz LoRa → Our counter: Hop to HF (shortwave, 3–30 MHz; not easily jammed) Directional antennas (Yagi, parabolic): • Concentrate RF energy in a 5–15° beam • Jammer must be in that beam to be effective • If jammer is NOT in beam, 20–30 dB of additional rejection Wired fallback (always): • batman-adv over Ethernet / dark fiber is completely immune to RF jamming • Every outdoor node should have a wired fallback path if possible

import asyncio
from kademlia.network import Server

BOOTSTRAP_NODES = [
    ("10.10.1.1", 5678),   # known-good node A
    ("10.10.1.2", 5678),   # known-good node B
    ("10.10.1.3", 5678),   # known-good node C
]

async def validated_get(key: str) -> str | None:
    """
    Query the DHT from 3 independent bootstrap points.
    Accept result only if >= 2 agree. Disagreement = possible Sybil attack.
    """
    results = []
    for bootstrap in BOOTSTRAP_NODES:
        node = Server()
        await node.listen(0)
        await node.bootstrap([bootstrap])
        val = await node.get(key)
        results.append(val)
        await node.stop()

    # Majority vote
    counts = {}
    for r in results:
        counts[r] = counts.get(r, 0) + 1

    winner = max(counts, key=counts.get)
    if counts[winner] >= 2:
        return winner
    else:
        print(f"⚠️  DHT inconsistency detected for key '{key}' — possible Sybil attack!")
        # Broadcast warning via Nostr #alert
        return None

Real-World Case Studies

These historical events show which tactics succeeded, which failed, and what the freedom network architecture is specifically designed to address.

🇮🇷 Iran — Green Revolution (2009)

SLA actions: Internet throttled to ~56 kbps, SMS blocked, proxy lists distributed then seized, Twitter/Facebook blocked at DPI level.

What failed: Centralised proxy lists were seized and turned into honeypots. Activists using those proxies were identified.

🇪🇬 Egypt — Arab Spring (2011)

SLA actions: Complete BGP withdrawal — Egypt went offline for 5 days (Jan 27–Feb 2). Four major ISPs withdrew their AS prefixes.

What worked: Dial-up modems via international lines (France, Italy); satellite; ham radio operators relayed status.

What failed: Everything that depended on any Egyptian ISP, including VPNs hosted inside Egypt.

Freedom network lesson: batman-adv + WiFi mesh continued to operate within cities even with full BGP withdrawal. APRS and Winlink HF provided inter-city comms.

🇧🇾 Belarus — Post-Election Protests (2020)

SLA actions: Internet blackouts for 2–3 days; Telegram monitored but not blocked (usage was too high); VPN traffic throttled; IMSI catchers deployed at protest locations.

🇲🇲 Myanmar — Military Coup (2021)

What failed: Standard OpenVPN and WireGuard protocols were blocked within days of the coup. SMS-based communications were trivially monitored.

Freedom network lesson: The nightly shutdown schedule would have been irrelevant to our batman-adv/Yggdrasil mesh — local services continued 24/7. The DPI blocking of VPN protocols validates the Yggdrasil-first design (Yggdrasil traffic looks like noise, not a VPN protocol).

The red team operates with limited knowledge (they know the network exists but do not have operator access) and reports findings to the full operator group.

The Core Principle: Make Disruption Expensive

No network is perfectly indestructible. The goal is not invulnerability — it is making the cost of sustained suppression economically and operationally prohibitive for even a well-resourced state adversary. Every layer of redundancy, every encrypted hop, every extra node they must seize, every extra jammer they must deploy — all of these raise the cost while the network continues to function at reduced capacity.

The adversary must disrupt continuously. The network only needs to survive long enough for the political situation to change. History consistently shows that communication infrastructure that survives authoritarian disruption plays a decisive role in the outcome.

The 1000-Node Rule Once a freedom network has more than ~1000 active nodes spread across a metropolitan area, physically suppressing it becomes operationally equivalent to suppressing every household simultaneously. At that scale, the network is effectively unstoppable by physical means alone, and the adversary must shift to the much harder problem of social and legal suppression — which the cryptographic and anonymity layers are specifically designed to resist.

Sending APRS Messages from the Freedom Network

We write a bridge daemon that watches for urgent alerts on the Nostr relay (events tagged #emergency) and re-broadcasts them as APRS messages via Dire Wolf's AGW interface — reaching every APRS receiver in the area even if the mesh is isolated.

📄 nostr_to_aprs.py — Emergency alert bridge
import asyncio, json, websockets, socket, struct

NOSTR_RELAY  = "ws://nostr-relay.mesh.local:7777"
DIREWOLF_AGW = ("localhost", 8000)  # Dire Wolf AGW port
MYCALL       = "FREEDOM1-9"
DESTCALL     = "APRS"

def agw_send_packet(sock, from_call: str, to_call: str, info: str):
    """Send an AX.25 UI frame via Dire Wolf's AGW TCP interface."""
    call_pad = lambda c: (c.ljust(10))[:10].encode()
    header = struct.pack("<BBBB10s10sII",
        0,           # port
        0, 0, 0,      # reserved
        ord('T'),    # frame type: transmit unproto
        0,           # PID reserved
        call_pad(from_call),
        call_pad(to_call),
        0,           # reserved
        len(info)
    )
    sock.sendall(header + info.encode())

async def watch_nostr():
    agw = socket.create_connection(DIREWOLF_AGW)
    async with websockets.connect(NOSTR_RELAY) as ws:
        await ws.send(json.dumps([
            "REQ", "emergency-sub",
            {"kinds": [1], "#t": ["emergency"], "limit": 0}
        ]))
        print("Watching Nostr for #emergency events...")
        async for raw in ws:
            msg = json.loads(raw)
            if msg[0] != "EVENT":
                continue
            event = msg[2]
            text = event["content"][:60]  # APRS comment max ~62 chars
            aprs_info = f":FREEDOMNET :ALERT {text}"
            agw_send_packet(agw, MYCALL, DESTCALL, aprs_info)
            print(f"[APRS TX] {aprs_info}")

asyncio.run(watch_nostr())

Automating Winlink as a Backup Transport

We write a watchdog that monitors the WireGuard backbone. If the tunnel is down for more than 10 minutes, it automatically queues critical updates (DHT peer lists, IPFS CIDs, Nostr relay list) as Winlink messages to all known far-city callsigns.

📄 winlink_watchdog.py — Backbone-down failover to HF email
import subprocess, time, json, os, datetime

WG_PEER       = "10.200.1.2"          # backbone peer to ping
OUTAGE_THRESH = 600                    # seconds before triggering HF failover
REMOTE_CALLS  = ["FREEDOM2", "FREEDOM3"]

last_seen = time.time()
hf_sent   = False

while True:
    ping = subprocess.run(["ping", "-c1", "-W2", WG_PEER], capture_output=True)
    if ping.returncode == 0:
        last_seen = time.time()
        hf_sent   = False
    elif time.time() - last_seen > OUTAGE_THRESH and not hf_sent:
        print(f"[{datetime.datetime.now():%H:%M}] Backbone DOWN — triggering HF failover")

        # Gather current state snapshot
        snapshot = {
            "timestamp": datetime.datetime.utcnow().isoformat(),
            "node":      "FREEDOM1",
            "status":   "backbone_down",
            "ipfs_pins": subprocess.getoutput("ipfs pin ls --type recursive").split(),
            "ygg_addr":  subprocess.getoutput("yggdrasilctl getSelf"),
        }
        with open("/tmp/freedom_snapshot.json", "w") as f:
            json.dump(snapshot, f, indent=2)

        # Send via Pat to all remote city nodes
        for callsign in REMOTE_CALLS:
            subprocess.run([
                "pat", "compose",
                "--to", f"{callsign}@winlink.org",
                "--subject", "BACKBONE DOWN — state snapshot",
                "--attachment", "/tmp/freedom_snapshot.json",
                "--body", f"Backbone to {WG_PEER} has been down {OUTAGE_THRESH//60}+ min."
            ])
        subprocess.run(["pat", "connect", "varahf:///"])
        hf_sent = True

    time.sleep(30)

DMR ↔ Matrix Voice Bridge

When a DMR operator transmits, the AMBE-encoded voice is forwarded to FreeDMR. A bridge bot decodes AMBE to PCM, transcodes to Opus, and forwards the audio as a voice message into a Matrix room — and vice versa. This means a Matrix/Element user on a phone in the mesh can hear and respond to DMR radio traffic.

📄 dmr_matrix_bridge.py — AMBE ↔ Opus ↔ Matrix voice
import asyncio, subprocess
from mautrix.client import AsyncClient
from mautrix.types import AudioInfo, MediaMessageEventContent

MATRIX_HOMESERVER = "http://matrix-srv.mesh.local:8448"
MATRIX_USER       = "@dmr-bridge:mesh.local"
MATRIX_TOKEN      = "syt_..."   # from ansible vault
MATRIX_ROOM       = "!dmr-radio:mesh.local"

async def bridge_dmr_to_matrix(ambe_frames: bytes):
    """
    Convert AMBE voice frames from DMR to Opus and post to Matrix as voice message.
    Uses ambe-audio (https://github.com/anyfusion/ambe-audio) for AMBE decode.
    """
    # Step 1: AMBE → PCM via ambe-audio or DSD+
    proc = subprocess.run(
        ["ambe-audio", "--decode", "--rate", "8000"],
        input=ambe_frames, capture_output=True
    )
    pcm = proc.stdout

    # Step 2: PCM → Opus via ffmpeg
    opus = subprocess.run(
        ["ffmpeg", "-f", "s16le", "-ar", "8000", "-ac", "1",
         "-i", "pipe:0", "-c:a", "libopus", "-f", "ogg", "pipe:1"],
        input=pcm, capture_output=True
    ).stdout

    # Step 3: Upload to Matrix and send as voice message
    client = AsyncClient(MATRIX_HOMESERVER, MATRIX_USER)
    client.access_token = MATRIX_TOKEN
    url = await client.upload_media(opus, mime_type="audio/ogg")

    await client.send_message_event(
        MATRIX_ROOM,
        "m.room.message",
        MediaMessageEventContent(
            msgtype="m.audio",
            url=url,
            body="DMR voice message",
            info=AudioInfo(mimetype="audio/ogg", duration=0)
        )
    )

DMR Encryption

DMR supports two encryption modes:

  • Basic Privacy (BP) — simple XOR scrambling, NOT cryptographically secure. Trivially broken.
  • Advanced Encryption Standard (AES-256) — full symmetric encryption. Requires all radios to share the same AES key. Available on AnyTone, Hytera, and Motorola Tier II/III radios.

Programme all freedom network DMR radios with the same AES-256 key (exchanged in-person during the node onboarding ceremony — Chapter 14). Rotate the key on the same schedule as WireGuard keys.

from prometheus_client import start_http_server, Gauge
import subprocess, re, time, json

# ── Gauges (no node-identifying labels) ──────────────────────
batman_neighbors   = Gauge("freedom_batman_neighbors_total",
                           "batman-adv direct neighbours visible")
ygg_peers          = Gauge("freedom_yggdrasil_peers_total",
                           "Yggdrasil peers connected")
wg_peers_ok        = Gauge("freedom_wireguard_peers_active",
                           "WireGuard peers with handshake < 3 min ago")
dht_routing_table  = Gauge("freedom_dht_routing_table_size",
                           "Kademlia routing table entries")
ipfs_peers         = Gauge("freedom_ipfs_peers_total",
                           "IPFS swarm peers connected")

def collect():
    # batman-adv neighbours
    out = subprocess.getoutput("batctl n 2>/dev/null")
    batman_neighbors.set(max(0, len(out.strip().splitlines()) - 2))

    # Yggdrasil peers
    try:
        raw = subprocess.getoutput("yggdrasilctl getPeers 2>/dev/null")
        data = json.loads(raw)
        ygg_peers.set(len(data.get("peers", {})))
    except Exception:
        ygg_peers.set(0)

    # WireGuard peers with recent handshake
    wg_out = subprocess.getoutput("wg show all latest-handshakes 2>/dev/null")
    now = time.time()
    active = sum(1 for line in wg_out.splitlines()
                 if len(line.split()) == 3 and
                 now - float(line.split()[2]) < 180)
    wg_peers_ok.set(active)

    # IPFS swarm peers
    try:
        raw = subprocess.getoutput("ipfs swarm peers --enc=json 2>/dev/null")
        ipfs_peers.set(len(json.loads(raw).get("Peers", [])))
    except Exception:
        ipfs_peers.set(0)

start_http_server(9200)
print("Mesh exporter running on :9200")
while True:
    collect()
    time.sleep(55)

Alert Bridge — Alertmanager → Matrix + Nostr

Alertmanager's webhook fires at our bridge daemon, which formats the alert and simultaneously posts it to a Matrix room and publishes it as a Nostr event tagged #alert — reachable by any node on the mesh regardless of which services are currently up.

📄 /opt/freedom/alert_bridge.py
from flask import Flask, request
import requests, json, time, hashlib, secrets
import secp256k1      # pip install secp256k1

app = Flask(__name__)

# ── Config (loaded from env / ansible vault) ──────────────────
MATRIX_URL   = "http://matrix-srv.mesh.local:8448"
MATRIX_TOKEN = "syt_..."
MATRIX_ROOM  = "!alerts:mesh.local"

NOSTR_RELAY  = "ws://nostr-relay.mesh.local:7777"
NOSTR_PRIVKEY = bytes.fromhex("...")  # alert bot's Nostr private key

def matrix_post(text: str):
    requests.post(
        f"{MATRIX_URL}/_matrix/client/v3/rooms/{MATRIX_ROOM}/send/m.room.message",
        headers={"Authorization": f"Bearer {MATRIX_TOKEN}"},
        json={"msgtype": "m.text", "body": text},
        timeout=5
    )

def nostr_publish(text: str, tags: list):
    """Build and sign a Nostr kind-1 event, publish to local relay."""
    pubkey = secp256k1.PrivateKey(NOSTR_PRIVKEY).pubkey.serialize()[1:].hex()
    created_at = int(time.time())
    event_data = json.dumps([0, pubkey, created_at, 1, tags, text],
                            separators=(',', ':'))
    event_id = hashlib.sha256(event_data.encode()).hexdigest()
    sig = secp256k1.PrivateKey(NOSTR_PRIVKEY).schnorr_sign(
        bytes.fromhex(event_id), bip340tag=None, raw=True).hex()
    event = {"id": event_id, "pubkey": pubkey, "created_at": created_at,
             "kind": 1, "tags": tags, "content": text, "sig": sig}
    import websockets, asyncio
    async def send():
        async with websockets.connect(NOSTR_RELAY) as ws:
            await ws.send(json.dumps(["EVENT", event]))
    asyncio.run(send())

@app.route("/alert", methods=["POST"])
def receive_alert():
    payload = request.json
    for alert in payload.get("alerts", []):
        status   = alert.get("status", "firing").upper()
        name     = alert["labels"].get("alertname", "Unknown")
        severity = alert["labels"].get("severity", "info").upper()
        summary  = alert.get("annotations", {}).get("summary", "")

        icon = "🔴" if severity == "CRITICAL" else "🟡" if severity == "WARNING" else "ℹ️"
        msg = f"{icon} [{status}] {name} ({severity})\n{summary}"

        try: matrix_post(msg)
        except Exception as e: print(f"Matrix error: {e}")

        try: nostr_publish(msg, [["t", "alert"], ["t", "freedom-network"]])
        except Exception as e: print(f"Nostr error: {e}")

    return "ok", 200

if __name__ == "__main__":
    app.run(host="127.0.0.1", port=9095)

Mesh DNS — Human-Readable Names Inside Yggdrasil

CoreDNS runs inside the Yggdrasil overlay, giving every service a .mesh hostname. Names are registered automatically via the Kademlia DHT. No public DNS is ever queried. No domain registrar can take your names down.

Why bother with names? Yggdrasil addresses like 200:aaa:bbb:1::1 are cryptographically derived and cannot be faked — but humans cannot remember them and configs quickly become unreadable. A .mesh zone resolves those addresses locally, with zero external dependency, making every config file and every user-facing URL human-readable without sacrificing the security of address-based routing.

Architecture

MESH DNS ARCHITECTURE ═══════════════════════════════════════════════════════════════════ Every node's /etc/resolv.conf: nameserver ::1 ← local stub resolver (systemd-resolved or unbound) search mesh.local Local stub → CoreDNS on mesh-router-1 (authoritative for .mesh) │ ┌─────────┴─────────┐ ▼ ▼ Zone data from Negative answers Kademlia DHT forwarded to (self-updating) upstream DoT/DoH (clearnet lookups, only when needed) Name registration flow: 1. Node starts, generates/loads Yggdrasil address 2. Node publishes dns:matrix-srv.mesh → 200:aaa:ccc:1::1 to DHT 3. CoreDNS plugin polls DHT every 60s, rebuilds zone 4. All other nodes' stub resolvers: matrix-srv.mesh → resolves instantly No central DNS server to seize. No registrar to coerce. No TTL games — records live as long as the DHT entry lives.

DHT-Driven Auto-Registration

Every node that starts a service publishes its hostname → Yggdrasil address mapping into the Kademlia DHT under the key prefix dns:. A daemon on the CoreDNS node polls the DHT every 60 seconds, collects all dns:* entries, and rewrites the zone file atomically. CoreDNS hot-reloads the zone automatically.

📄 /opt/freedom/dns_registrar.py — each node self-registers its name
import asyncio, json, subprocess, time
from kademlia.network import Server

BOOTSTRAP  = [("10.10.1.1", 5678)]
HOSTNAME   = "matrix-srv"          # this node's .mesh name (set per node)
SERVICES   = ["matrix", "nostr"]   # informational tags

def get_ygg_addr() -> str:
    raw = subprocess.getoutput('yggdrasilctl getSelf 2>/dev/null | python3 -c \
        "import sys,json; d=json.load(sys.stdin); print(d[\'address\'])"')
    return raw.strip()

async def register_loop():
    while True:
        addr = get_ygg_addr()
        if not addr:
            print("Yggdrasil not ready, retrying...")
            await asyncio.sleep(10)
            continue

        node = Server()
        await node.listen(0)
        await node.bootstrap(BOOTSTRAP)

        record = json.dumps({
            "addr":     addr,
            "hostname": HOSTNAME,
            "services": SERVICES,
            "ts":       int(time.time()),
        })
        await node.set(f"dns:{HOSTNAME}.mesh", record)
        print(f"[DNS] Registered {HOSTNAME}.mesh → {addr}")
        await node.stop()

        await asyncio.sleep(55)   # re-publish before DHT TTL expires

asyncio.run(register_loop())
📄 /opt/freedom/dns_zone_builder.py — CoreDNS node: poll DHT → rebuild zone
import asyncio, json, time, os, tempfile, shutil
from kademlia.network import Server

BOOTSTRAP  = [("10.10.1.1", 5678)]
ZONE_FILE  = "/var/lib/coredns/mesh.zone"
ZONE_ORIGIN = "mesh"
KNOWN_NAMES = [  # scan all known hostnames; extend as network grows
    "matrix-srv", "nostr-relay", "asterisk", "ipfs-node",
    "xmpp-srv", "monitor-vm",
    "mesh-router-1", "mesh-router-2", "mesh-router-3",
    "vpn-gateway", "i2p-router", "reticulum-node", "sdr-node",
]

def build_zone(records: dict) -> str:
    serial = int(time.time())
    ns1_addr = records.get("mesh-router-1", "::1")
    lines = [
        f"$ORIGIN {ZONE_ORIGIN}.",
        "$TTL 60",
        f"@   IN SOA  ns1.{ZONE_ORIGIN}. ops.{ZONE_ORIGIN}. ({serial} 60 30 600 60)",
        f"    IN NS   ns1.{ZONE_ORIGIN}.",
        f"ns1 IN AAAA {ns1_addr}",
        "",
    ]
    for name, addr in sorted(records.items()):
        lines.append(f"{name:<20} IN AAAA {addr}")
    return "\n".join(lines) + "\n"

async def sync_loop():
    while True:
        node = Server()
        await node.listen(0)
        await node.bootstrap(BOOTSTRAP)

        records = {}
        for hostname in KNOWN_NAMES:
            raw = await node.get(f"dns:{hostname}.mesh")
            if raw:
                try:
                    data = json.loads(raw)
                    # Reject stale entries (older than 5 minutes)
                    if time.time() - data["ts"] < 300:
                        records[hostname] = data["addr"]
                except Exception:
                    pass

        await node.stop()

        if records:
            zone_content = build_zone(records)
            # Atomic write — CoreDNS reload never sees a partial file
            tmp = ZONE_FILE + ".tmp"
            with open(tmp, "w") as f:
                f.write(zone_content)
            shutil.move(tmp, ZONE_FILE)
            print(f"[DNS] Zone updated: {len(records)} records")

        await asyncio.sleep(60)

asyncio.run(sync_loop())

DNSSEC signs all zone records with the CoreDNS node's key, so any resolving node can verify that a DNS response was not tampered with in transit. This closes a Sybil attack vector where a rogue node lies about the address of matrix-srv.mesh.

OLSR — Optimized Link State Routing

OLSR (RFC 3626 / OLSRv2 RFC 7181) is a proactive link-state routing protocol designed for mobile ad-hoc networks. Where batman-adv works at Layer 2 (frames), OLSR works at Layer 3 (IP routing) — making it complementary to batman-adv and a strong choice for larger, more geographically distributed meshes.

How OLSR Works

HELLO Messages — Neighbour Discovery

Every OLSR node broadcasts HELLO messages periodically (default every 2 seconds). Each HELLO lists the node's neighbours and their link quality. Neighbours use this to build a local topology view and to elect MPRs.

MPR — Multipoint Relays (the key efficiency innovation)

In a naive link-state protocol, every node floods every topology change to the entire network — O(n²) messages. OLSR uses Multipoint Relays: each node selects a minimal subset of its neighbours that covers all 2-hop neighbours. Only MPRs re-flood Topology Control (TC) messages. This reduces flood overhead dramatically — typically by 60–80% in dense meshes.

MPR SELECTION — HOW OLSR PRUNES FLOODING Without MPR (full flood — every node re-broadcasts): [A]→[B]→[C]→[D]→[E]→[F] AND [A]→[C]→[B]→[D]→[E]→[F] AND (n! paths) [A]→[B]→[D]→[C]→[E]→[F] ... With MPR (only MPR-selected nodes re-broadcast TC): Node A selects B and D as MPRs (they together cover all of A's 2-hop neighbours) [A] sends TC → [B] and [D] re-broadcast → done [C], [E], [F] receive but do NOT re-broadcast Result: same topology information reaches all nodes with 60-80% fewer transmissions

TC Messages — Topology Control

MPR nodes broadcast TC (Topology Control) messages advertising which nodes have selected them as MPRs. All nodes collect TC messages and build a full link-state topology map. Dijkstra's shortest-path algorithm computes the optimal next-hop for each destination — this is written into the kernel routing table.

OLSR Metrics and Link Quality

OLSRv2 supports pluggable link metrics. The default is ETX (Expected Transmission Count) — a measure of link quality derived from HELLO packet loss. A perfect link has ETX = 1.0; a 50% packet-loss link has ETX = 4.0. OLSR routes traffic via the minimum-ETX path, naturally avoiding lossy links.

Tinc — The Self-Healing Mesh VPN

Tinc is a VPN daemon that automatically discovers the shortest direct path between any two nodes, bypassing the central server when possible. If the central server disappears, all nodes that can still reach each other directly continue to function. No other VPN has this property — WireGuard, OpenVPN, and Nebula all require at least one always-on relay when direct connectivity fails.

TINC MESH ROUTING — AUTO-SHORTCUTTING Topology defined via config: mesh-router-1 ── vpn-gateway ── mesh-router-2 │ mesh-router-3 When mesh-router-1 and mesh-router-2 can reach each other directly: Tinc detects this via its meta-protocol and opens a DIRECT encrypted tunnel. Traffic bypasses vpn-gateway entirely. When vpn-gateway goes down: Direct paths already exist — no re-convergence needed. mesh-router-3 uses mesh-router-1 or mesh-router-2 as relays automatically. Result: no SPOF. Any node can relay for any other node.

Stealth Layer 1 — Shadowsocks: Encrypted SOCKS5 Proxy

Shadowsocks wraps any TCP traffic in a lightweight AEAD-encrypted stream with no fixed header. The server behaves like a high-entropy random-byte source — DPI engines cannot classify it. Originally developed to bypass the Great Firewall of China; widely battle-tested under active DPI inspection.

Stealth Layer 3 — Meek: Domain Fronting over CDN

Domain fronting routes traffic through a CDN (Cloudflare, Fastly, Azure CDN) using a front domain in the SNI field and the real destination in the HTTP Host header. The network sees a connection to cloudflare.com; the CDN internally forwards to your server. Blocking it requires blocking the entire CDN — which is economically prohibitive for most states.

Stealth Layer 4 — Port Knocking: Invisible Listeners

A port-knocking daemon keeps all VPN ports closed to uninitiated connections. To "open" the port, an operator sends a specific sequence of TCP SYN packets to decoy ports in order. A passive scanner scanning the server sees all ports closed. An adversary with a port scanner cannot even discover the VPN is running.

📄 knockd — single-packet authentication for WireGuard

[options]
    interface   = ens18
    logfile     = /dev/null           # no logs

Stealth Layer 5 — Traffic Shaping and Timing Obfuscation

Even perfectly encrypted traffic leaks information through packet timing, size, and volume. A state adversary watching upstream bandwidth can often infer "VPN active" even without breaking encryption. Traffic shaping adds random delay and padding to defeat this — at the cost of some latency.

📄 tc — add jitter and constant-bitrate shaping to wg0
# Pad all WireGuard packets to a fixed size (hides packet length variation)
# and add random 0–50ms delay (defeats timing correlation)

# Create a root queueing discipline on wg0
tc qdisc add dev wg0 root handle 1: prio

# Add netem (network emulator) for artificial jitter
tc qdisc add dev wg0 parent 1:1 handle 10: netem \
  delay 25ms 25ms distribution normal   # mean 25ms, stddev 25ms jitter

# View current shaping rules:
tc qdisc show dev wg0

# NFQUEUE-based packet padder (Python — pads all packets to nearest 100-byte boundary)
pip3 install NetfilterQueue
cat > /opt/freedom/pad_packets.py <<'EOF'
from netfilterqueue import NetfilterQueue
from scapy.all import IP
import random, time

JITTER_MAX = 0.050   # 50ms max jitter
PAD_BLOCK   = 100    # round to nearest 100-byte block

def process(pkt):
    time.sleep(random.uniform(0, JITTER_MAX))
    data = bytearray(pkt.get_payload())
    remainder = len(data) % PAD_BLOCK
    if remainder:
        data += bytes(PAD_BLOCK - remainder)   # zero-pad
    pkt.set_payload(bytes(data))
    pkt.accept()

nfq = NetfilterQueue()
nfq.bind(0, process)
nfq.run()
EOF

# Route WireGuard output through NFQUEUE:
nft add rule inet filter output oif wg0 queue num 0

import json, os, statistics
from collections import Counter
from datetime import datetime, timedelta

CONN_LOG = "/usr/local/zeek/logs/current/conn.log"

def hourly_connection_counts() -> dict:
    counts = Counter()
    with open(CONN_LOG) as f:
        for line in f:
            if line.startswith("#"): continue
            cols = line.split("\t")
            src = cols[2] if len(cols) > 2 else "?"
            counts[src] += 1
    return counts

def detect_anomaly(current: Counter, baseline: list[Counter]):
    for src, count in current.items():
        historical = [b.get(src, 0) for b in baseline]
        if not historical: continue
        mean = statistics.mean(historical)
        stdev = statistics.stdev(historical) if len(historical) > 1 else mean
        if count > mean + 3 * stdev:
            print(f"ANOMALY: {src} made {count} connections "
                  f"(mean={mean:.0f}, 3σ={mean+3*stdev:.0f})")

"""Pull Nostr notes from the mesh relay and push to YaCy index."""
import asyncio, websockets, json, requests, time

RELAY    = "ws://[200:aaa:bbb:140::1]:7777"
YACY_API = "http://[200:aaa:bbb:133::1]:8090/api/index.json"
AUTH     = ("admin", "PASSWORD")

async def index_nostr():
    async with websockets.connect(RELAY) as ws:
        await ws.send(json.dumps(["REQ", "idx-1", {"kinds": [1], "limit": 200}]))
        async for raw in ws:
            msg = json.loads(raw)
            if msg[0] != "EVENT": continue
            event = msg[2]
            doc = {
                "url":  f"nostr:{event['id']}",
                "title": f"Nostr note by {event['pubkey'][:8]}",
                "text":  event["content"],
                "date":  time.strftime("%Y%m%d", time.gmtime(event["created_at"])),
            }

Matrix Room Indexer — Opt-In Public Room Search

📄 matrix-yacy-indexer.py — index public Matrix rooms
#!/usr/bin/env python3
"""Index public Matrix room messages into YaCy."""
import asyncio, nio, requests, time

MATRIX = "https://matrix.mesh.local:8448"
TOKEN  = "syt_bot_ACCESS_TOKEN"
YACY   = "http://[200:aaa:bbb:133::1]:8090/api/index.json"
AUTH   = ("admin", "PASSWORD")

async def run():
    client = nio.AsyncClient(MATRIX, "@searchbot:mesh.local")
    client.access_token = TOKEN
    resp = await client.joined_rooms()
    for room_id in resp.rooms:
        msgs = await client.room_messages(room_id, start="", limit=500)
        for event in msgs.chunk:
            if not isinstance(event, nio.RoomMessageText): continue
            requests.post(YACY, auth=AUTH, json={
                "url":   f"matrix:{room_id}/{event.event_id}",
                "title": f"Matrix: {room_id}",
                "text":  event.body,
                "date":  time.strftime("%Y%m%d",
                          time.gmtime(event.server_timestamp / 1000)),
            }, timeout=5)
    await client.close()

asyncio.run(run())

Real-World Physical Layers

From fiber to long-range radio — the physical infrastructure of a people's internet.

Fiber

Community fiber networks (often called "Community Fiber to the Home" or CFTH) connect neighborhoods using dark fiber — optical cables that carry no ISP traffic, privately owned and operated. Once lit, fiber provides symmetric gigabit speeds over kilometers with no radio interference concerns and near-zero latency.

  • Use MikroTik or Ubiquiti media converters to bring fiber into standard Ethernet
  • Connect to batman-adv Ethernet interfaces directly
  • Bury conduit and pull fiber — communities have done this for <$1000/mile in rural areas

WiFi — 802.11s Mesh Mode

Modern WiFi adapters (especially those based on ath9k, ath10k, mt76) support 802.11s — a native mesh extension to the WiFi standard. It is more efficient than ad-hoc mode for larger meshes and integrates with batman-adv.

📄 Configure 802.11s mesh on real hardware

SDR — Software-Defined Radio

SDR (using HackRF, RTL-SDR, LimeSDR, or USRP) allows custom radio waveforms entirely in software. For a freedom network:

  • Transmit on unlicensed ISM bands (915 MHz, 2.4 GHz, 5.8 GHz) using GNU Radio flowgraphs
  • Implement custom spread-spectrum waveforms difficult to detect or jam
  • Use FreqShow or GQRX to monitor spectrum for interference and jammers
  • License-free digital voice: FreeDV (codec2 over HF/VHF) for VOIP over SDR

LoRa / LoRaWAN

LoRa (Long Range) operates in the sub-GHz ISM band and achieves 2–20 km range at very low power (10 mW) but low data rate (250 bps – 11 kbps). Ideal for:

  • Text messages (Nostr events, XMPP messages fit in LoRa packets)
  • Routing protocol messages (batman-adv OGMs, Yggdrasil peering)
  • Sensor data, emergency alerts
  • Long-distance relay between mesh islands where WiFi can't reach

Defense in Depth

No single technology solves all threats. The freedom network achieves resilience through layers: physical diversity (fiber + WiFi + LoRa), routing diversity (batman-adv + Yggdrasil), application diversity (Matrix + XMPP + Nostr), and anonymity diversity (I2P + Hyphanet + WireGuard). Removing any one layer degrades but does not destroy the network.

Startup Order

When bringing up the network from scratch, start layers from the bottom up:

  1. Physical / Ethernet links (Proxmox bridge, real WiFi, LoRa modems)
  2. batman-adv on all mesh routers
  3. Yggdrasil on all mesh routers (auto-discovers via batman-adv multicast)
  4. WireGuard on vpn-gateway (connects remote sites)
  5. GNU Radio DSSS/FHSS flowgraphs + TUN bridges (SDR nodes)
  6. Reticulum daemon on all nodes (picks up all available interfaces)
  7. DHT bootstrap node on mesh-router-1 (announces services)
  8. IPFS (content backbone — announces itself to DHT on start)
  9. I2P (anonymity layer for services)
  10. Matrix, XMPP, Nostr, Asterisk (announce to DHT on start)
  11. Hyphanet last (takes longest to integrate into the network)
